Logic phân loại
TierTrangConcurrencyPath

Output: VolumeProfile
tier → S / M / L / XL
page_count → số trang thực
estimated_spans → tổng số span ước tính
estimated_tokens → tokens × giá Gemini
estimated_cost_usd → chi phí dự kiến

estimated_duration_min → thời gian ước tính
Cost Estimator (tích hợp trong Volume Detector)
📏 page_count × avg_spans_per_page × 📝 avg_tokens_per_span × quality_multiplier × 💰 gemini_price_per_1M_tokens = Estimated cost USD
quality_multiplier: fast=1× · balanced=2× · high=3× (3 Gemini calls)

③ Quality Tier Engine

fast

⚡ 1-phase

1 Gemini call/block
→ Draft translation only
→ Best for L/XL docs
→ ~3× cheaper + faster
balanced ← default M

⚖️ 2-phase

2 Gemini calls/block
→ Draft + verify
→ Good quality/cost ratio
→ ~2× cheaper than high
high ← default S

🎯 3-phase (current)

3 Gemini calls/block
→ Draft + glossary + verify
→ Best cho doc quan trọng
→ Giữ nguyên pipeline hiện tại
User có thể override quality tier bất kể volume — VD: doc lớn nhưng muốn chất lượng cao, hoặc doc nhỏ nhưng chỉ cần fast.

⑥ Resume Capability

Luồng resume
1. Job bị interrupt (mất mạng / crash)
2. Page.status = failed hoặc translating stuck
3. User gọi POST /resume
4. System tìm pages chưa compiled
5. Re-queue từ đúng stage còn thiếu
6. Giữ nguyên translations đã dịch xong
Resume logic chi tiết
raw → cần extract lại
extracted → cần translate từ đầu
translating (stuck) → re-queue translate
translated → chỉ cần compile
failed → retry từ stage fail
compiled → skip (đã xong)
Timeout "stuck": job chạy > 30 phút không cập nhật → tự động đánh dấu failed
