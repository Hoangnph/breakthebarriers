Tóm lại, function calling cung cấp quyền truy cập trực tiếp tới một vài function cụ thể, trong khi MCP
là framework giao tiếp chuẩn hóa cho phép LLM khám phá và sử dụng một phạm vi rất rộng
các tài nguyên bên ngoài. Với ứng dụng đơn giản, các tool cụ thể là đủ; với
hệ thống AI phức tạp, có nhiều kết nối và cần thích nghi, một chuẩn phổ quát như MCP
là thiết yếu.
Các cân nhắc bổ sung cho MCP
Dù MCP đưa ra một framework mạnh mẽ, việc đánh giá kỹ lưỡng đòi hỏi
cân nhắc một số khía cạnh quan trọng ảnh hưởng tới mức độ phù hợp của nó với từng use case.
Hãy xem kỹ hơn một số khía cạnh:
Tool vs. Resource vs. Prompt: Cần hiểu rõ vai trò cụ thể
của các thành phần này. Resource là dữ liệu tĩnh (ví dụ, file PDF, record trong database).
Tool là một function có thể thực thi để thực hiện action (ví dụ, gửi
email, truy vấn API). Prompt là template hướng dẫn LLM cách
tương tác với resource hoặc tool, đảm bảo tương tác có cấu trúc và
hiệu quả.
Discoverability: Một lợi thế chính của MCP là MCP client có thể truy vấn động
server để biết nó cung cấp những tool và resource nào. Cơ chế khám phá "just-in-time"
này rất mạnh với agent cần thích nghi với các năng lực mới
mà không cần redeploy.
Security: Expose tool và dữ liệu qua bất kỳ protocol nào cũng cần các biện pháp bảo mật
vững chắc. Một implementation MCP phải bao gồm authentication và
authorization để kiểm soát client nào được truy cập server nào và
những action cụ thể nào chúng được phép thực hiện.
Implementation: Dù MCP là một chuẩn mở, implementation của nó có thể
phức tạp. Tuy vậy, các provider đang bắt đầu đơn giản hóa quy trình này. Ví dụ,
một số model provider như Anthropic hoặc FastMCP cung cấp SDK
trừu tượng hóa phần lớn boilerplate code, giúp developer dễ dàng hơn khi
tạo và kết nối MCP client và server.
Error Handling: Một chiến lược xử lý lỗi toàn diện là rất quan trọng. Protocol
phải định nghĩa cách các lỗi (ví dụ, tool execution thất bại, server không khả dụng,
request không hợp lệ) được truyền ngược về LLM để nó hiểu
thất bại và có thể thử một cách tiếp cận khác.
Local vs. Remote Server: MCP server có thể được triển khai cục bộ trên cùng
máy với agent hoặc từ xa trên server khác. Server cục bộ có thể được
chọn vì tốc độ và bảo mật với dữ liệu nhạy cảm, trong khi server từ xa
