Trong ví dụ này, chúng ta đang sử dụng LLM để phê bình và nâng cao một prompt khác. Tương tác
cấp meta này minh họa tính linh hoạt và sức mạnh của các mô hình này,
cho phép chúng ta xây dựng các hệ thống agentic hiệu quả hơn bằng cách trước tiên tối ưu hóa
các hướng dẫn cơ bản mà chúng nhận được. Đây là một vòng lặp hấp dẫn nơi AI giúp chúng ta nói chuyện với AI tốt hơn.
Prompting cho Các Nhiệm Vụ Cụ Thể
Trong khi các kỹ thuật được thảo luận cho đến nay có thể áp dụng rộng rãi, một số nhiệm vụ được hưởng lợi từ
các cân nhắc prompting cụ thể. Chúng đặc biệt liên quan trong lĩnh vực code và đầu vào đa phương thức.
Prompting Code
Các mô hình ngôn ngữ, đặc biệt là những mô hình được đào tạo trên các tập dữ liệu code lớn, có thể là những trợ lý mạnh mẽ
cho các nhà phát triển. Prompting cho code bao gồm sử dụng LLM để tạo ra,
giải thích, dịch hoặc gỡ lỗi code. Nhiều trường hợp sử dụng tồn tại:
Prompt để viết code: Yêu cầu mô hình tạo ra các đoạn code hoặc
các hàm dựa trên mô tả về chức năng mong muốn.
Ví dụ: "Viết một hàm Python lấy danh sách các số và trả về
giá trị trung bình."
Prompt để giải thích code: Cung cấp một đoạn code và yêu cầu mô hình
giải thích nó làm gì, từng dòng hoặc trong tóm tắt.
Ví dụ: "Giải thích đoạn code JavaScript sau: [chèn code]."
Prompt để dịch code: Yêu cầu mô hình dịch code từ một
ngôn ngữ lập trình sang ngôn ngữ khác.
Ví dụ: "Dịch code Java sau sang C++: [chèn code]."
Prompt để gỡ lỗi và xem xét code: Cung cấp code có lỗi
hoặc có thể được cải thiện và yêu cầu mô hình xác định vấn đề, đề xuất sửa lỗi, hoặc
đề xuất tái cấu trúc.
Ví dụ: "Code Python sau đang gây ra 'NameError'. Có gì sai
và làm thế nào tôi có thể sửa nó? [chèn code và traceback]."
Prompting code hiệu quả thường đòi hỏi cung cấp đủ ngữ cảnh, chỉ định
ngôn ngữ và phiên bản mong muốn, và rõ ràng về chức năng hoặc vấn đề.
