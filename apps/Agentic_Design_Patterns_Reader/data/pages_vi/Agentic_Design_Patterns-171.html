On-demand vs. Batch: MCP có thể hỗ trợ cả session tương tác theo nhu cầu
lẫn xử lý batch quy mô lớn hơn. Lựa chọn phụ thuộc vào
ứng dụng, từ một conversational agent thời gian thực cần truy cập tool
ngay lập tức tới một pipeline phân tích dữ liệu xử lý record theo batch.
Transportation Mechanism: Protocol cũng định nghĩa các tầng transport nền
cho giao tiếp. Với tương tác cục bộ, nó dùng JSON-RPC
qua STDIO (standard input/output) để giao tiếp liên tiến trình hiệu quả.
Với kết nối từ xa, nó tận dụng các protocol thân thiện với web như Streamable
HTTP và Server-Sent Events (SSE) để hỗ trợ giao tiếp client-server
bền vững và hiệu quả.
Model Context Protocol dùng mô hình client-server để chuẩn hóa luồng thông tin.
Hiểu tương tác giữa các thành phần là chìa khóa cho hành vi agentic nâng cao của MCP
:
1. Large Language Model (LLM): Trí tuệ lõi. Nó xử lý request của người dùng,
lập kế hoạch, và quyết định khi nào cần truy cập thông tin bên ngoài
hoặc thực hiện một action.
2. MCP Client: Đây là một ứng dụng hoặc wrapper quanh LLM. Nó đóng vai trò
trung gian, chuyển intent của LLM thành một request chính thức tuân thủ
chuẩn MCP. Nó chịu trách nhiệm khám phá, kết nối tới, và
giao tiếp với MCP Server.
3. MCP Server: Đây là cổng vào thế giới bên ngoài. Nó expose một tập tool,
resource, và prompt cho bất kỳ MCP Client được ủy quyền nào. Mỗi server thường
chịu trách nhiệm cho một domain cụ thể, chẳng hạn kết nối tới database nội bộ
của công ty, một email service, hoặc một public API.
4. Optional Third-Party (3P) Service: Đây là tool, ứng dụng,
hoặc nguồn dữ liệu bên ngoài thực sự mà MCP Server quản lý và expose. Nó là
endpoint cuối cùng thực hiện action được yêu cầu, chẳng hạn truy vấn một
database proprietary, tương tác với một nền tảng SaaS, hoặc gọi một public
weather API.
Luồng tương tác diễn ra như sau:
1. Discovery: MCP Client, thay mặt LLM, truy vấn một MCP Server để
hỏi nó cung cấp những năng lực nào. Server phản hồi bằng một manifest liệt kê
các tool có sẵn (ví dụ, send_email), resource (ví dụ, customer_database), và
prompt.
