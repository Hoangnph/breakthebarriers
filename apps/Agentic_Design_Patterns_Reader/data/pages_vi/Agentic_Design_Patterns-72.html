Lịch sử hội thoại được duy trì và truyền vào mô hình ngôn ngữ ở mỗi
bước để cung cấp ngữ cảnh cho cả giai đoạn tạo/tinh chỉnh và reflection. Cuối cùng,
script in ra phiên bản code được tạo cuối cùng sau khi vòng lặp kết thúc.
Ví dụ code thực hành (ADK)
Bây giờ hãy xem một ví dụ code mang tính khái niệm được triển khai bằng Google ADK.
Cụ thể, code minh họa điều này bằng cách dùng cấu trúc Generator-Critic,
trong đó một thành phần (Generator) tạo ra kết quả hoặc kế hoạch ban đầu, còn một
thành phần khác (Critic) cung cấp phản hồi phê bình hoặc phần critique, dẫn dắt Generator
đến một đầu ra cuối cùng tinh chỉnh hơn hoặc chính xác hơn.
from google.adk.agents import SequentialAgent, LlmAgent
# Agent đầu tiên tạo bản nháp ban đầu.
generator = LlmAgent(
name="DraftWriter",
description="Tạo nội dung bản nháp ban đầu về một chủ đề cho trước.",
instruction="Viết một đoạn văn ngắn, giàu thông tin về
chủ đề của người dùng.",
output_key="draft_text" # Đầu ra được lưu vào state key này.
)
# Agent thứ hai phê bình bản nháp từ agent thứ nhất.

Bạn là một người kiểm chứng sự thật tỉ mỉ.
