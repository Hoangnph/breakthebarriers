Cần tạo file __init__.py trong thư mục fastmcp_client_agent. Việc này
đảm bảo agent được ADK nhận diện như một Python package có thể discovery.
Để bắt đầu, mở terminal mới và chạy `python fastmcp_server.py` để khởi động
FastMCP server. Tiếp theo, đi tới thư mục cha của `fastmcp_client_agent` (ví dụ,
`adk_agent_samples`) trong terminal và thực thi `adk web`. Khi
ADK Web UI tải trong trình duyệt, hãy chọn `fastmcp_greeter_agent` từ menu agent
. Sau đó bạn có thể kiểm thử bằng cách nhập prompt như "Greet John Doe." Agent sẽ
dùng tool `greet` trên FastMCP server của bạn để tạo phản hồi.
Tổng quan nhanh
What: Để hoạt động như agent hiệu quả, LLM phải vượt ra ngoài việc sinh văn bản
đơn giản. Chúng cần khả năng tương tác với môi trường bên ngoài để truy cập
dữ liệu hiện tại và sử dụng phần mềm bên ngoài. Nếu không có phương thức giao tiếp chuẩn hóa,
mỗi tích hợp giữa LLM và tool hoặc nguồn dữ liệu bên ngoài
trở thành một nỗ lực tùy chỉnh, phức tạp, và khó tái sử dụng. Cách làm ad-hoc này cản trở
khả năng mở rộng và khiến việc xây dựng các hệ thống AI phức tạp, có nhiều kết nối trở nên khó khăn và
kém hiệu quả.
Why: Model Context Protocol (MCP) đưa ra một giải pháp chuẩn hóa bằng cách đóng vai trò như
một interface phổ quát giữa LLM và hệ thống bên ngoài. Nó thiết lập một protocol mở,
được chuẩn hóa, định nghĩa cách các năng lực bên ngoài được khám phá và sử dụng.
Hoạt động theo mô hình client-server, MCP cho phép server expose tool, data
resource, và prompt tương tác cho bất kỳ client tuân thủ chuẩn nào. Các ứng dụng
được LLM hỗ trợ đóng vai trò những client này, khám phá động và tương tác với các resource có sẵn
theo cách dễ dự đoán. Cách tiếp cận chuẩn hóa này nuôi dưỡng một hệ sinh thái gồm
các component interoperable và reusable, đơn giản hóa mạnh mẽ việc phát triển
các agentic workflow phức tạp.
Rule of thumb: Dùng Model Context Protocol (MCP) khi xây dựng các hệ thống agentic phức tạp,
có khả năng mở rộng, hoặc cấp enterprise, cần tương tác với một tập hợp đa dạng và
đang thay đổi gồm tool, nguồn dữ liệu, và API bên ngoài. Nó lý tưởng khi interoperability
giữa các LLM và tool khác nhau là ưu tiên, và khi agent cần khả năng
khám phá động năng lực mới mà không cần redeploy. Với các ứng dụng đơn giản hơn
có số lượng function định nghĩa sẵn cố định và hạn chế, direct tool
function calling có thể là đủ.
