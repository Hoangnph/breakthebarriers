2. Các Quy tắc Mobile-First di động tối cao (Apple & Android Standard)

Frontend được xây dựng từ thiết bị di động trước để kiểm soát độ chật hẹp, tránh giật lag phần cứng rồi sau đó mở rộng cho màn hình lớn Desktop.

Vùng an toàn (Notch & Safe Area padding)

Cưỡng chế CSS sử dụng biến môi trường safe-area-inset. Tiêu đề chính, Hamburger menu và thanh điều phối Reader không bao giờ bị cắt khuất bởi Notch (tai thỏ) của iPhone hay Home Indicator (thanh vuốt trang chủ):

padding-top: env(safe-area-inset-top);
padding-bottom: env(safe-area-inset-bottom);

Kích thước vùng chạm (W3C Touch Target)

Tất cả các phần tử điều khiển (Chuyển trang, Phóng to/Thu nhỏ, Toggle ngôn ngữ, Hamburger menu) có kích thước chạm tối thiểu **$44\text{px} \times 44\text{px}$** và đệm an toàn **8px** xung quanh để ngón tay dễ thao tác mà không bấm nhầm.

Trọng lượng nhẹ & Phản hồi < 100ms

Loại bỏ hoàn toàn các frameworks nặng như React/NextJS hay thư viện cuộn. Sử dụng cơ chế Iframe Sandboxing cô lập CSS tài liệu gốc, JS DOM thao tác trực tiếp đạt độ trễ vẽ (render latency) gần như bằng **0ms**.

1. Thuật toán tịnh tiến chống đè chữ LSE (Layout Shift Engine)

Giải quyết rào cản tiếng Việt dài ra gây đè dòng. Chạy động ngay khi nạp trang trong iframe để tính toán chiều rộng và chiều cao bao (Bounding boxes) của từng thẻ span:

/**
 * GIẢI THUẬT LAYOUT SHIFT ENGINE (CLIENT-SIDE RUNTIME)
 * Tự động tính chiều cao tăng thêm của các thẻ span khi xuống dòng và tịnh tiến các phần tử phía dưới.
 */
function runLayoutShiftEngine(iframeDoc) {
    // Lấy tất cả spans định vị pixel tuyệt đối của Poppler
    const spans = Array.from(iframeDoc.querySelectorAll('span[data-span-id]'));
    
    // 1. Sắp xếp spans theo tọa độ top trước, left sau để tạo dòng đọc tuyến tính
    spans.sort((a, b) => {
        const topA = parseFloat(a.style.top);
        const topB = parseFloat(b.style.top);
        if (Math.abs(topA - topB) < 5) {
            return parseFloat(a.style.left) - parseFloat(b.style.left);
        }
        return topA - topB;
    });

    let accumulatedShift = 0;
    
    spans.forEach((span) => {
        // Lưu trữ tọa độ gốc để tránh bị cộng lũy kế sai lệch khi resize màn hình
        const originalTop = parseFloat(span.getAttribute('data-original-top') || span.style.top);
        if (!span.getAttribute('data-original-top')) {
            span.setAttribute('data-original-top', originalTop);
        }
        
        // Cân chỉnh vị trí dựa trên độ dời tích lũy từ các dòng trước
        const currentTop = originalTop + accumulatedShift;
        span.style.top = `${currentTop}px`;
        
        // Đo đạc chiều cao thực tế (wrapped) và chiều cao gốc không xuống dòng (clone unwrapped)
        const measureClone = span.cloneNode(true);
        measureClone.style.whiteSpace = 'nowrap';
        measureClone.style.visibility = 'hidden';
        measureClone.style.position = 'absolute';
        span.parentNode.appendChild(measureClone);
        
        const unwrappedHeight = measureClone.getBoundingClientRect().height;
        const wrappedHeight = span.getBoundingClientRect().height;
        span.parentNode.removeChild(measureClone);
        
        const deltaHeight = wrappedHeight - unwrappedHeight;
        
        // Nếu phát hiện dồn dòng (> 3px lệch), cộng lũy kế cho các spans tiếp theo
        if (deltaHeight > 3) {
            accumulatedShift += deltaHeight;
        }
    });
}

2. Thuật toán Autoscale & Touch Gesture

Autoscale Viewport Math

Đảm bảo nội dung trang định vị pixel tuyệt đối (thường rộng 900px-1188px) luôn khớp chính xác với màn hình di động mà không gây cuộn ngang:

const viewportWidth = window.innerWidth;
const pageCanonicalWidth = 900; // Chiều rộng chuẩn của Poppler HTML
const scaleFactor = viewportWidth / pageCanonicalWidth;

// Áp dụng ma trận tỉ lệ lên trang trong iframe
iframe.style.transform = `scale(${scaleFactor})`;
iframe.style.transformOrigin = "top left";
iframe.style.width = `${pageCanonicalWidth}px`;
iframe.style.height = `${window.innerHeight / scaleFactor}px`;

Swipe 3D Gesture Matrix

Bắt cử chỉ vuốt ngang trên di động, cập nhật lớp biến đổi CSS để lật trang như đọc sách thật:

let touchStartX = 0;
let touchEndX = 0;

document.addEventListener('touchstart', e => {
    touchStartX = e.changedTouches[0].screenX;
});

document.addEventListener('touchend', e => {
    touchEndX = e.changedTouches[0].screenX;
    handleSwipeGesture();
});

function handleSwipeGesture() {
    const threshold = 80; // pixel tối thiểu
    if (touchStartX - touchEndX > threshold) {
        // Vuốt sang trái -> Trang kế tiếp
        nextPageWithFlipAnimation();
    } else if (touchEndX - touchStartX > threshold) {
        // Vuốt sang phải -> Trang trước đó
        prevPageWithFlipAnimation();
    }
}

Giai đoạn 4: Thuật toán Autoscale Engine & Responsive Iframe

Thời lượng: 1 ngày
Tiêu chí hoàn thành (DoD):
  • Khi xoay ngang/dọc điện thoại, iframe tự động scale tương thích, không sinh thanh cuộn ngang khó chịu.

Giai đoạn 5: Triển khai thuật toán Layout Shift Engine (LSE) & Font Shrink

Thời lượng: 1 ngày
Tiêu chí hoàn thành (DoD):

Giai đoạn 6: Tích hợp Bộ điều khiển Translation Memory & Hot Re-compile

Thời lượng: 1 ngày
Tiêu chí hoàn thành (DoD):
  • Biên tập viên dễ dàng tra cứu, sửa đổi bản dịch của bất kỳ thẻ span nào.
  • Hiệu quả biên dịch cập nhật lên iframe tức thời mà không cần reload trang.

1. Quy trình Kiểm định & Xác thực (Verification Plan)

Để đảm bảo tệp tĩnh và các logic cốt lõi của Frontend hoạt động bền bỉ, không lỗi DOM hay đổ vỡ giao diện trên di động, hệ thống sử dụng quy trình kiểm tra 2 lớp:

A. Kiểm thử Tự động hóa (Automated E2E Tests)

Cấu trúc lại bộ kiểm tra run_e2e_automation_tests.py để tự động quét các tệp tin Frontend mới:

  • Xác thực cấu trúc cú pháp DOM HTML (DOM Integrity).
  • Kiểm định tính toàn vẹn của liên kết tĩnh (Broken Link Checker).
  • Đảm bảo các tệp tĩnh CSS/JS nội bộ (CDN-Free) tải thành công 100%.
python run_e2e_automation_tests.py

B. Kiểm thử Thủ công (Manual & Responsive Tests)

Khởi chạy máy chủ dịch vụ tĩnh để mô phỏng và kiểm tra thực tế trên di động:

  • Giả lập Chrome DevTools Device Mode chạy các dòng máy iPhone 14 Pro, Samsung Galaxy S20.
  • Kiểm tra Touch Swipe lật trang mượt mà < 150ms.
  • Xoay ngang màn hình di động xác nhận Autoscale co dãn iframe tự động.
  • Toggle chế độ song ngữ, rà soát LSE chống chồng chữ.

2. Các Quyết định Thiết kế & Đồng thuận kỹ thuật

1. Kích hoạt Dịch thuật Tự động (Auto-Translate)

Hệ thống tự động kích hoạt API /translate bất đồng bộ ngay sau khi quá trình /extract tệp PDF hoàn tất thành công. Việc chạy ngầm non-blocking này tối ưu hóa tối đa thao tác của người sử dụng.

2. Chiều rộng tiêu chuẩn (900px Standard Width)

Cưỡng chế cố định kích thước 900px làm chiều rộng tiêu chuẩn của trang HTML xuất ra từ Poppler CLI. Đây là kích thước chuẩn hóa tối ưu giúp Autoscale Engine tính toán tỷ lệ thu phóng chính xác tuyệt đối, triệt tiêu hoàn toàn compound scale factor error.
