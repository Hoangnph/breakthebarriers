2. Các thành phần cốt lõi của SA

Touch Gesture Engine

Bắt các sự kiện touchstart và touchend trên di động. Tính toán hướng và gia tốc vuốt để lật trang tự nhiên bằng hiệu ứng 3D CSS.

Layout Shift Engine (LSE)

Tự động xử lý dồn dịch dòng chữ tiếng Việt dịch dài hơn tiếng Anh bằng cách thay đổi thuộc tính top động của cây DOM trong iframe, tránh chồng chéo.

Mobile Autoscale Engine

Tự động tính toán tỷ lệ chiều ngang di động và áp dụng transform: scale() lên container chứa trang sách để hiển thị vừa khít mọi kích cỡ màn hình.

Safeguard Pipeline Engine

Hệ thống tiền lọc Encoding/DOM Validator tĩnh và 3 trạm kiểm soát chất lượng (Quality Gates). Tự động kích hoạt cơ chế bẫy lỗi (Paragraph Fallback, Syntax JSON Repair, Dynamic Font Shrink) bảo đảm hệ thống hoạt động tin cậy 100%.

High-Fidelity Translation Engine (HFE)

Bộ gom spans theo tọa độ cận kề để tái tạo ngữ cảnh cho AI, rà soát dịch đa bước (Thô -> Đánh bóng -> Đối chiếu), kết hợp Glossary-Enforced cưỡng chế từ điển chuyên ngành giữ 100% ngữ nghĩa nguyên bản.

Local State Sync

Đồng bộ tức thì mọi cài đặt (ngôn ngữ, độ thu phóng, chương sách, theme tối/sáng) vào bộ nhớ localStorage để lưu giữ trạng thái đọc của người dùng.

Quy trình Xử lý & Bẫy lỗi Tài liệu tĩnh (Processing Safeguard Pipeline)

Để đảm bảo việc thông dịch hàng loạt 482 trang sách diễn ra an toàn, bảo toàn cấu trúc tuyệt đối và hiển thị hoàn hảo trên di động, hệ thống áp dụng quy trình kiểm soát chất lượng 3 lớp và cơ chế tự phục hồi lỗi xử lý:

1

A. Quy trình Vòng đời & Quản lý Thư mục (Document Management Workflow)

Quy trình di chuyển dữ liệu tĩnh qua 6 pha khép kín để chạy offline hiệu quả:

  • Pha Ingestion (Nhập liệu): Sách PDF kỹ thuật gốc được lưu trữ tại data/raw_pdf/.
  • Pha Extraction (Trích xuất): Sử dụng pdftohtml xuất các trang absolute sang data/extracted_html/.
  • Pha Sanitization (Chuẩn hóa): Chuẩn hóa mã hóa ký tự sang UTF-8 tĩnh và lọc bỏ màu nền xám thô `#A0A0A0` về `#FFFFFF`.
  • Pha Localization Mapping (Bản địa hóa): Beautiful Soup bóc tách text span lưu thành JSON bản địa tiếng Anh (`data/locale/en/`) ánh xạ khóa-giá trị 1-1.
  • Pha Batch Translation (Dịch hàng loạt): Gộp các đoạn text ngăn cách bởi `
    ` dịch qua Google Translate API, ghi nhận tại `data/locale/vi/`.
  • Pha Compilation (Biên dịch tĩnh): Regex Injector tái dựng trang HTML song ngữ lưu tại `data/pages/`, ghi nhận manifest.
2

B. Các Trạm Kiểm soát Chất lượng (Quality Gates)

Hệ thống thiết lập 3 chốt kiểm soát tự động để ngăn ngừa tài liệu hỏng:

  • Quality Gate 1 (Sanitizer Check): Tự động sửa đổi charset tĩnh về UTF-8 và chèn thẻ meta charset để tránh lỗi hiển thị dấu tiếng Việt.
  • Quality Gate 2 (DOM Integrity Gate): So khớp số lượng thẻ <span> giữa trang gốc và trang dịch. Nếu lệch nhau, hệ thống lập tức báo lỗi và khóa chặn quá trình xuất bản.
  • Quality Gate 3 (Kiểm soát đè chữ - Overlap Gate): Quét tọa độ inline `top`/`left` để phát hiện các dòng có xu hướng đè lấn nhau khi dồn chữ Việt.
3

C. Quy trình Bẫy lỗi & Phục hồi Động (Error Trapping & Auto-Recovery)

Chính sách tự sửa lỗi và phục hồi được tích hợp ở cả tầng biên dịch và client-side:

  • Google Mismatch Fallback: Nếu API dịch làm mất thẻ `
    `, gây lệch chỉ mục, hệ thống tự động bẫy lỗi và fallback sang Dịch đơn lẻ từng đoạn (Paragraph Fallback).
  • JSON Syntax Auto-Repair: Sửa nhanh các lỗi định dạng cú pháp JSON kinh điển bằng Regex. Nếu irreparable, tự động fallback hiển thị chữ tiếng Anh gốc kèm thẻ cảnh báo [?] để tránh treo ứng dụng.
  • LSE Canvas Overflow Control: Nếu chữ tiếng Việt quá dài đẩy chiều cao trang vượt quá canvas $1188\text{px}$, client-side sẽ kích hoạt Dynamic Font Shrink co 5% - 10% cỡ chữ hoặc line-clamping để bảo toàn khung trang di động.
4

D. Cơ chế Thông dịch Chất lượng cao (High-Fidelity Translation)

Đảm bảo việc thông dịch trôi chảy và bảo toàn 100% ý nghĩa văn bản gốc:

  • Context-Aware Reconstruction: Gom các thẻ span cận kề để tái tạo đoạn văn bản đầy đủ ngữ cảnh cho AI, gắn thẻ chỉ mục mã hóa ẩn để giữ nguyên ánh xạ 1-1.
  • Agentic Multi-Stage Translation: Dịch đa bước (Dịch thô -> Đánh bóng văn phong kỹ thuật -> Đối chiếu song hành phát hiện sai lệch ngữ nghĩa).
  • Glossary-Enforced Injections: Cưỡng chế áp dụng bộ từ điển chuyên ngành cốt lõi, ngăn chặn việc dịch sai hoặc địa phương hóa quá mức làm mất giá trị tài liệu kỹ thuật.

2. Thuật toán Cân chỉnh Dồn Dòng (LSE Formula)

Công thức tính tịnh tiến vị trí `top` cho từng khối văn bản khi dịch từ Anh sang Việt:

Biến số đầu vào:
d_i : div_index (Danh sách div đã sắp xếp theo top tăng dần)
original_top(d_i) : Tọa độ top ban đầu trích xuất từ PDF gốc
accumulatedShift_i : Độ dời tích lũy tính đến div thứ i

Công thức:
new_top(d_i) = original_top(d_i) + accumulatedShift_i

Nếu d_i xảy ra hiện tượng dồn dòng (wrapped):
unwrapped_height = Chiều cao đo được khi white-space: nowrap
wrapped_height = Chiều cao đo được khi white-space: normal
delta_height = wrapped_height - unwrapped_height

Nếu delta_height > 2px:
  accumulatedShift_(i+1) = accumulatedShift_i + delta_height
Ngược lại:
  accumulatedShift_(i+1) = accumulatedShift_i

3. Trình Dịch thuật Chất lượng cao (HFE Engine Specs)

Đặc tả kỹ thuật của 3 cấu phần ngữ nghĩa HFE đảm bảo độ chính xác tuyệt đối:

A. Tái lập đoạn văn bản (Context Reconstruction)

def reconstruct_context_and_index(spans_list, threshold_y=5.0):
    """
    Gom cụm các span nằm trên cùng dòng (y-coordinate lệch < threshold_y)
    để tạo câu đầy đủ ngữ cảnh cho AI và chèn tag ẩn [s:idx].
    """
    # Sắp xếp các span theo tọa độ top, sau đó left
    spans_list.sort(key=lambda x: (x['top'], x['left']))
    groups = []
    current_group = []
    
    for span in spans_list:
        if not current_group:
            current_group.append(span)
        else:
            # So sánh độ lệch y của span hiện tại với span đầu tiên trong nhóm
            if abs(span['top'] - current_group[0]['top']) < threshold_y:
                current_group.append(span)
            else:
                groups.append(current_group)
                current_group = [span]
    if current_group:
        groups.append(current_group)
        
    reconstructed_blocks = []
    for g_idx, group in enumerate(groups):
        # Sắp xếp lại theo left để đúng thứ tự chữ từ trái qua phải
        group.sort(key=lambda x: x['left'])
        text_parts = []
        for s in group:
            text_parts.append(f"[s:{s['index']}] {s['text']}")
        reconstructed_blocks.append(" ".join(text_parts))
    return reconstructed_blocks

4. Cơ chế Bẫy lỗi & Quality Gates (Safeguard Specs)

Thông số kỹ thuật và logic bẫy lỗi ở các trạm kiểm soát chất lượng:

A. Điều kiện thông qua Trạm (Quality Gate Verification)

Gate 1: Encoding Check
  check_utf8(file) -> True/False
  Lọc bỏ CSS pdftohtml rác: re.sub(r'#a0a0a0', '#ffffff', css)

Gate 2: DOM Integrity Gate
  len(source_spans) == len(target_spans)
  Nếu lệch chỉ mục: Kích hoạt API Mismatch Fallback.

Gate 3: Overlap Gate
  For i in range(len(spans)-1):
    If spans[i+1].top == spans[i].top:
      Width_i = spans[i].left + spans[i].width
      If Width_i > spans[i+1].left:
        Raise OverlapWarning(span[i], span[i+1])

B. Thuật toán Phục hồi (Auto-Recovery Algorithms)

Algorithm 1: Mismatch API Recovery
  Input: translated_block (API gộp bằng <hr> bị mất thẻ phân tách)
  Action:
    1. Phát hiện: length_mismatch == True
    2. Fallback: Hủy bỏ lô gộp, split thành các đoạn văn riêng lẻ.
    3. Trả về: Dịch đơn lẻ từng đoạn một (Paragraph Fallback).

Algorithm 2: Client Canvas Limit (Dynamic Font Shrink JS Client-side)
  Input: container_height = 1188px
  Mã nguồn JavaScript đo đạc bounding-box thực tế và co dãn chữ chống tràn trang di động:
  ```javascript
  function adjustFontSizeToFit(iframe, maxHeight = 1188, step = 0.02, minScale = 0.75) {
      const doc = iframe.contentDocument || iframe.contentWindow.document;
      const body = doc.body;
      
      if (!body.style.fontSize) {
          body.style.fontSize = "100%";
      }
      
      let currentScale = 1.0;
      
      // Đo đạc tọa độ bottom lớn nhất thực tế của toàn bộ các thẻ absolute
      function getAbsoluteBoundingBottom() {
          let maxBottom = 0;
          const elements = body.querySelectorAll('div, span, p');
          elements.forEach(el => {
              const rect = el.getBoundingClientRect();
              const offsetTop = el.offsetTop || 0;
              const height = el.offsetHeight || rect.height || 0;
              const bottom = offsetTop + height;
              if (bottom > maxBottom) {
                  maxBottom = bottom;
              }
          });
          return maxBottom;
      }
      
      let realBottom = getAbsoluteBoundingBottom();
      
      // Co chữ động tuần tự nếu chiều cao vượt ngưỡng cho phép
      while (realBottom > maxHeight && currentScale > minScale) {
          currentScale -= step;
          body.style.fontSize = `${currentScale * 100}%`;
          
          // Chạy lại thuật toán LSE để tái tính toán các dịch chuyển top
          if (typeof window.applyLayoutShiftEngine === 'function') {
              window.applyLayoutShiftEngine(doc);
          }
          
          realBottom = getAbsoluteBoundingBottom();
      }
      
      return {
          finalBottom: realBottom,
          fittedScale: currentScale,
          overflowRecovered: realBottom <= maxHeight
      };
  }
  ```
