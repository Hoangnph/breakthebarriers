Động cơ Backend & Tài liệu API Tương tác

Hồ sơ đặc tả chi tiết kiến trúc Backend API-First dựa trên FastAPI, Pydantic và mô hình TDD (Test-Driven Development). Cho phép thử nghiệm trực tiếp, mô phỏng phản hồi thời gian thực và phân tích thuật toán dịch thuật chất lượng cao.

1. Cấu hình Môi trường ảo (.venv)

Hệ thống tuân thủ nghiêm ngặt nguyên lý cách ly môi trường để tránh xung đột thư viện hệ thống. Chúng tôi xây dựng môi trường ảo ổn định bằng Python 3.12.

Khắc phục lỗi liên kết thư viện Mac OS: Khi sử dụng Homebrew Python 3.14, xảy ra lỗi runtime pyexpat link library do thiếu ký hiệu _XML_SetAllocTrackerActivationThreshold trong hệ thống libexpat.1.dylib. Backend đã được chuyển dịch hoàn toàn sang Python 3.12.8 ổn định để bảo đảm 100% tính tương thích thư viện nhị phân.

2. Kiến trúc Cơ sở dữ liệu PostgreSQL

Hệ thống chuyển đổi từ lưu trữ dữ liệu phẳng mô phỏng sang PostgreSQL thực tế qua SQLAlchemy ORM để quản lý toàn diện tiến trình số hóa, đảm bảo tính toàn vẹn dữ liệu qua cơ chế Cascade Delete và giảm thiểu chi phí dịch lặp lại.

Cơ chế tự tạo bảng tự động (Metadata Create All): Khi máy chủ FastAPI khởi động, hệ thống sẽ tự động gọi Base.metadata.create_all(bind=engine). Nếu các bảng CSDL chưa tồn tại trong PostgreSQL, chúng sẽ được sinh tự động chính xác theo định nghĩa của SQLAlchemy ORM Models, giảm thiểu tối đa các bước thiết lập thủ công.

Chuỗi Xử lý Dịch thuật Giữ nguyên Layout tuyệt đối

Luồng xử lý backend bao gồm 3 cấu phần dịch vụ hoạt động theo mô hình đường ống (pipeline), đảm bảo tính toàn vẹn bố cục pixel và chất lượng dịch thuật ngữ nghĩa cao nhất.

app/services/extractor.py

Extractor (Lớp Tiền xử lý & Trích xuất)

Đảm nhận vai trò phân tích mã nguồn HTML thô tạo ra bởi pdftohtml, làm sạch tài nguyên và trích xuất tọa độ tuyệt đối của tất cả các phần tử văn bản.

Các tính năng kỹ thuật chính:

  • Lọc màu nền thô (Background Sanitization): Tự động tìm và thay thế mã màu nền thám hiểm #A0A0A0 thành màu trắng #FFFFFF thông qua Regular Expression hiệu năng cao, đem lại trải nghiệm đọc dễ chịu.
  • Bảo đảm Encoding: Tự động phát hiện và chèn thẻ meta charset <meta charset="utf-8"> vào ngay sau thẻ <head> của HTML nếu chưa có, ngăn ngừa triệt để lỗi hiển thị font tiếng Việt (UTF-8).
  • Tọa độ Span (Coordinate Extraction): Sử dụng BeautifulSoup4 duyệt toàn bộ các thẻ <span> có style định vị tuyệt đối để tách tọa độ top và left (kiểu float) lưu thành danh sách JSON chuẩn phục vụ dịch.

Thử nghiệm thuật toán Extractor:

app/services/translator.py

Translator (Thuật toán Dịch thuật ngữ nghĩa chất lượng cao)

Giải quyết bài toán cực khó của PDF: tài liệu được lưu dưới dạng các nhãn span bị cắt vụn ngẫu nhiên. Trình dịch sẽ tái cấu trúc ngữ cảnh trước khi đưa qua AI để giữ trọn vẹn ý nghĩa văn bản gốc.

Chi tiết kỹ thuật 3 bước cốt lõi:

  • Gom nhóm Spans theo dòng (Context Reconstruction): Tự động gom nhóm các nhãn span nằm trên cùng một hàng ngang (chênh lệch tọa độ đứng top nhỏ hơn một ngưỡng dung sai threshold_y = 5.0px). Việc này ghép các chữ đơn lẻ thành dòng chữ có nghĩa.
  • Sắp xếp Trái-qua-Phải (Horizontal Sort): Sắp xếp các span trong cùng dòng theo thứ tự tăng dần của tọa độ ngang left để giữ đúng thứ tự từ ngữ.
  • Mã hóa Ẩn (Token Interpolation): Thay vì ghép chuỗi thông thường, các span kế tiếp được nối lại bằng nhãn token phân tách [s:id]. AI sẽ dịch cả câu mà không làm mất dấu các thẻ ID này, cho phép tiêm bản dịch chính xác vào từng span gốc.
  • Glossary Overrides: Áp dụng Glossary (từ điển thuật ngữ chuyên ngành công nghệ) ở bước hậu xử lý để thay thế chính xác các từ đặc thù (ví dụ: Programming -> Lập trình).

Thử nghiệm thuật toán Gom Spans & Dịch:

 }
app/services/compiler.py

Compiler (Lớp Biên dịch, Đóng gói & Chống đè chữ)

Chịu trách nhiệm tái cấu trúc tài liệu tĩnh cuối cùng bằng cách thay thế nội dung dịch thuật vào giao diện và nhúng cơ chế bảo vệ hiển thị.

Quy trình an toàn:

  • Quality Gate 2 (DOM Integrity Checking): Hệ thống thực hiện rà soát nghiêm ngặt: Kiểm tra số lượng thẻ span có ID trong tệp HTML gốc và so sánh với danh sách khóa dịch nhận về. Nếu phát hiện thiếu bất kỳ phần tử nào (mất span), giao dịch biên dịch sẽ bị hủy bỏ ngay để tránh làm hỏng cấu trúc trang.
  • Nhúng mã Dynamic Font Shrink: Khi dịch từ tiếng Anh sang tiếng Việt, chiều dài ký tự thường tăng 20-30%, gây tràn lề hoặc đè chữ. Compiler tự động tiêm một script Javascript tối ưu ở cuối thẻ </body>. Khi trang tải, script sẽ ước lượng hộp giới hạn (Bounding Box Approx) của các phần tử và tự động co nhỏ font (tỷ lệ 0.75 - 1.0) để đảm bảo không một từ nào vượt ra ngoài canvas pixel chuẩn 1188px.

Mã bảo vệ Dynamic Font Shrink tự động tiêm:

window.addEventListener('load', function() {
     const pageWidthLimit = 1188; // Giới hạn kích thước Canvas chuẩn
     const absoluteSpans = document.querySelectorAll('span[style*="left"]');
     let maximumRightCoordinate = 0;
     
     absoluteSpans.forEach(span => {
         const leftCoord = parseFloat(span.style.left) || 0;
         const approxWidth = span.textContent.length * 8; // Ước lượng chiều ngang ký tự
         const rightCoord = leftCoord + approxWidth;
         if (rightCoord > maximumRightCoordinate) maximumRightCoordinate = rightCoord;
     });
     
     if (maximumRightCoordinate > pageWidthLimit) {
         const shrinkFactor = Math.max(0.75, Math.min(1.0, pageWidthLimit / maximumRightCoordinate));
         document.body.style.fontSize = `${shrinkFactor * 100}%`; // Co dãn thông minh
         console.log(`[Dynamic Font Shrink] Co nhỏ font còn ${shrinkFactor}`);
     }
 });

Nâng cấp Hệ thống Cấp Doanh nghiệp (Enterprise Upgrades)

Để đưa hệ thống từ trạng thái giả lập (Mock) lên một dịch vụ phân tích dịch thuật tài liệu cấp doanh nghiệp (Production-Grade), chúng tôi đã hoàn thiện 5 nâng cấp cốt lõi:

1. Tích hợp pdftohtml CLI Poppler trực tiếp

Gọi trực tiếp tiến trình hệ thống ở runtime:

  • Dynamic CLI Execution: Sử dụng subprocess.run gọi CLI pdftohtml của Poppler để số hóa PDF thành HTML tuyệt đối và trích xuất các asset ảnh gốc ngay khi người dùng upload.
  • Robust TDD Fallback: Tự động nhận diện môi trường kiểm thử để nạp dữ liệu Mock nhằm đảm bảo test suite luôn chạy qua 100%.

2. Kết nối Google Gemini API (Real SDK)

Tích hợp SDK google-generativeai chuyên sâu:

  • Real Gemini 1.5 Flash Model: Dịch thuật agentic với mô hình Gemini thực tế khi có biến môi trường GEMINI_API_KEY.
  • Chuỗi Prompts 3 Bước (HFE): Triển khai quy trình Dịch thô (Draft) -> Hiệu đính thuật ngữ (Refine với Glossary) -> Xác thực cấu trúc (Verify) đảm bảo tính toàn vẹn DOM và ngữ nghĩa.

3. Tác vụ Bất đồng bộ Non-blocking

FastAPI BackgroundTasks siêu nhẹ:

  • Async Mode Flag: Cung cấp tham số async_mode=True cho các đầu việc nặng như extract, translate, compile.
  • Tránh Client Timeout: Phản hồi ngay 202 Accepted kèm trạng thái xử lý nền (extracting, translating, compiling) cho phép frontend cập nhật thanh tiến trình mượt mà.

4. REST API Xóa Cascade & Tẩy rửa Tệp

Dọn dẹp triệt để ổ đĩa và database:

  • Cascade Database Deletion: Ràng buộc khóa ngoại SQLAlchemy cascade="all, delete-orphan" tự động xóa sạch documents, pages, translations chỉ với 1 query.
  • File Cleanups: Tự động dọn dẹp các tệp PDF gốc, thư mục HTML trích xuất và thư mục trang dịch hoàn chỉnh trên ổ đĩa vật lý.

5. Quản lý Bộ nhớ Dịch (Translation Memory)

Biên tập trực tiếp & Tự động tái biên dịch:

  • Search & Paginate: API tìm kiếm nhanh không phân biệt hoa thường (ilike) trên toàn bộ đệm dịch, phân trang với limit & offset.
  • Live Edit & Auto Re-compile: Khi cập nhật bản dịch qua PUT, backend tự động phát hiện và biên dịch lại ngay lập tức trang chứa span đó, cập nhật tức thì giao diện người đọc.

1. Phương pháp phát triển TDD (Test-Driven Development)

Để đảm bảo chất lượng phần mềm không bao giờ bị suy giảm (regression) trong suốt quá trình nâng cấp, backend được viết theo chu trình nghiêm ngặt: Red -> Green -> Refactor.

1. Viết Test Trước (RED)

Viết bộ unit test mô tả chính xác yêu cầu kỹ thuật của API và thuật toán trước khi lập trình service logic.

2. Viết Code Đạt Test (GREEN)

Viết mã nguồn tối giản nhất có thể để vượt qua tất cả các bài kiểm tra (Pass tests 100%).

3. Tối ưu hóa (REFACTOR)

Tiến hành tái cấu trúc, làm sạch mã nguồn, loại bỏ trùng lặp (DRY) trong khi vẫn đảm bảo bộ test luôn có màu xanh lá.

2. Bộ E2E Automation Testing Suite toàn diện

Để đảm bảo sự hoàn thiện 100% không bỏ sót bất kỳ lỗi hồi quy hay phá vỡ DOM nào, chúng tôi đã xây dựng bộ E2E Automation Testing Suite chuyên sâu tại run_e2e_automation_tests.py. Bộ test thực hiện rà soát tự động qua 4 tầng kiểm định độc lập.

Tầng 1: System & Environment

Kiểm tra phiên bản Python 3.12, môi trường ảo venv, cấu hình DATABASE_URL và kiểm tra ping kết nối trực tiếp PostgreSQL database thật.

Tầng 2: Pytest Suite

Tự động khởi chạy bộ 17/17 unit tests cho core services và API endpoint, sử dụng SQLite in-memory cô lập đảm bảo tốc độ cực cao (0.35s) và an toàn dữ liệu.

Tầng 3: DOM & Link Validation

Phân tích cú pháp DOM HTML tĩnh của tất cả 4 trang tài liệu hướng dẫn bằng Parser tùy biến để phát hiện thẻ hở, thẻ đóng sai lệch, liên kết hỏng (Broken link) và tài nguyên tĩnh bị mất.

Tầng 4: E2E API Flow Simulation

Khởi tạo FastAPI TestClient đa luồng mô phỏng hoàn chỉnh luồng nghiệp vụ: Upload PDF -> Extract HTML -> Translate Spans -> Compile Pages, tự động xác nhận CSDL và chặn lỗi Quality Gate 2.

Câu lệnh kích hoạt và Chạy Automation Test Suite

Bạn có thể dễ dàng chạy bộ kiểm thử toàn diện này từ thư mục gốc của dự án bằng cách kích hoạt môi trường ảo và chạy script kiểm thử Python:
