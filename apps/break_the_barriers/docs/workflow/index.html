1. Chuẩn bị Tài nguyên & API-First Backend

Để đáp ứng nhu cầu mở rộng linh hoạt, hệ thống kết hợp hoàn hảo giữa **quy trình chuẩn bị tài nguyên tĩnh (Offline Prep)** và **động cơ Backend API-First**. Để tuân thủ triệt để triết lý YAGNI (You Aren't Gonna Need It) và DRY (Don't Repeat Yourself), hệ thống sử dụng cơ sở dữ liệu PostgreSQL thực tế để theo dõi trạng thái và lưu bộ đệm thông dịch động, thay vì chạy các chuyển đổi nhị phân nặng nề lúc runtime:

  • Quản lý Trạng thái qua PostgreSQL (PostgreSQL State Hub): Toàn bộ vòng đời số hóa và thông dịch được lưu trữ và đồng bộ hóa qua 3 bảng dữ liệu quan hệ, giúp tránh việc dịch thuật lặp lại và dễ dàng phục hồi khi có sự cố.
  • Cấu trúc Thư mục Lưu trữ Khoa học (Static & Symlink Layout): Tổ chức lưu trữ tài nguyên tĩnh tách biệt rõ ràng thông qua liên kết Symlink từ data/ trỏ đến shared resources, giúp tối ưu dung lượng đĩa và độc lập hóa asset DRM.
  • Developer-first API Orchestration: Nhà phát triển và client dễ dàng tương tác qua hệ thống API RESTful phong phú của FastAPI, tự động khởi tạo bảng tự động (Metadata Create All) và sẵn sàng cho các quy trình biên dịch hàng loạt.

2. Chuyển đổi và bảo toàn định dạng gốc

Khác với các công cụ đọc text thông thường làm mất đi sơ đồ thiết kế đồ họa, hệ thống sử dụng thuật toán số hóa chuyển đổi tài liệu sang HTML với cơ chế định vị tọa độ tuyệt đối (Absolute Positioning). Để tuân thủ nghiêm ngặt triết lý YAGNI (You Aren't Gonna Need It) và DRY (Don't Repeat Yourself), hệ thống sử dụng kiến trúc biên dịch tĩnh siêu nhẹ thay vì tích hợp các bộ giải mã nặng nề:

1. Tiền biên dịch HTML Tĩnh (Poppler pdftohtml)

Áp dụng triết lý YAGNI - Loại bỏ gánh nặng tính toán runtime:

  • Offline Preserving: Sử dụng công cụ biên dịch hiệu năng cao pdftohtml (thuộc bộ tiện ích Poppler) để chuyển đổi PDF tĩnh một lần duy nhất sang HTML định vị tuyệt đối từng pixel.
  • Zero-JS Client Overhead: Không tích hợp các thư viện giải mã PDF nhị phân khổng lồ (như PDF.js) vào trình duyệt của người dùng, giúp giảm dung lượng tải trang và tăng tốc thời gian render lên gấp 10 lần.
  • Asset Inheritance: Kế thừa trực tiếp các file ảnh nền và cấu trúc CSS absolute gốc được sinh ra tĩnh.

2. Bóc tách Cấu trúc DOM Tinh gọn & Trạm Kiểm soát (BS4 - Quality Control)

Áp dụng triết lý DRY và kiểm soát chất lượng chặt chẽ:

  • BeautifulSoup (bs4) Parser: Kịch bản Python extract_to_json.py duyệt cây DOM của HTML gốc để trích xuất nội dung văn bản từ các thẻ <span> thành tệp tin JSON bản địa (English) theo cơ chế index 1-1.
  • Trạm Xác thực Cấu trúc (DOM Validator Check): Tự động so khớp số lượng thẻ span giữa tệp locale Việt và Anh để đảm bảo không bị dịch thiếu dòng, sót chữ trước khi biên dịch ngược lại HTML.
  • Bộ kiểm soát mã hóa (Encoding Sanitizer): Phát hiện tệp rỗng, chuẩn hóa mã ký tự từ ISO-8859-1 sang UTF-8 tĩnh để hiển thị dấu tiếng Việt hoàn hảo không lỗi font.

3. Đồng bộ Local hóa Động (Regex Rebuild Engine)

Hiệu năng tối cao - Tự động tái dựng trang song ngữ:

  • Regex HTML Generator: Kịch bản build_translated_html.py sử dụng các biểu thức chính quy (Regular Expressions) siêu tốc để tìm kiếm và tráo đổi văn bản tiếng Anh sang tiếng Việt tĩnh, sinh ra các trang pages_vi/ tương ứng.
  • Iframe Swap Runtime: Phía client-side chỉ cần hoán đổi thuộc tính src của thẻ <iframe> khi người dùng chuyển đổi ngôn ngữ (VIE/ENG) thay vì chạy code JavaScript dịch và chèn thẻ DOM phức tạp.
  • Accented Charset Fix: Tự động hiệu chỉnh mã ký tự (charset) từ ISO-8859-1 sang UTF-8 tĩnh để hiển thị hoàn hảo dấu tiếng Việt.

3. AI Thông dịch & Hệ thống Bẫy lỗi (Safeguard Pipeline)

Để đảm bảo quá trình dịch thuật 482 trang sách diễn ra tin cậy 100%, bảo toàn bố cục pixel gốc và hiển thị mượt mà trên di động, hệ thống sử dụng một **Quy trình Quản lý, Kiểm soát Chất lượng & Bẫy lỗi khép kín (Safeguard Pipeline)**:

1. Hệ thống CSDL PostgreSQL & 6 Pha Di chuyển Dữ liệu Kỹ thuật

Để thay thế kiến trúc quản lý tệp tĩnh thủ công bằng một giải pháp tin cậy cấp doanh nghiệp, toàn bộ trạng thái tài liệu được quản trị qua **PostgreSQL Hub** kết hợp với REST API của FastAPI:

  • Cơ sở Dữ liệu Quan hệ PostgreSQL:
    • Bảng documents: Lưu trữ metadata tài liệu, tổng số trang, và trạng thái quy trình (raw, extracted, translated, compiled).
    • Bảng pages: Lưu trữ mã HTML gốc (original_html) và HTML đã biên dịch dịch song ngữ (translated_html), ràng buộc Cascade Delete với bảng documents.
    • Bảng translations: Lưu trữ đệm từng thẻ <span> dịch thuật độc lập (span_id, original_text, translated_text), giúp tránh dịch trùng lặp và giảm tải chi phí API (DRY).
  • 6 Pha Di chuyển Dữ liệu (Backend API Pipeline):
    • Pha 1 (Upload): Gọi POST /api/docs/upload để tải PDF lên, tạo record trong bảng documents với trạng thái ban đầu raw.
    • Pha 2 (Extraction & Initial Mapping): Gọi POST /api/docs/{doc_id}/extract, backend kích hoạt pdftohtml và parser BeautifulSoup để điền dữ liệu gốc vào bảng pages và từng thẻ span vào bảng translations, chuyển trạng thái tài liệu thành extracted.
    • Pha 3 (Sanitization): Tự động làm sạch các thẻ lỗi và chuyển bảng mã sang UTF-8 động trước khi cập nhật cơ sở dữ liệu.
    • Pha 4 (AI Translation): Gọi POST /api/docs/{doc_id}/translate, hệ thống kích hoạt chuỗi đại lý dịch thuật thông minh 3 giai đoạn (HFE) để dịch spans và lưu vào bảng translations.
    • Pha 5 (Regex Reconstruction): Gọi POST /api/docs/{doc_id}/compile, Regex Engine biên dịch động lại trang bằng cách tráo nội dung thẻ span, nhúng cơ chế Co dãn Font chống đè chữ (Dynamic Font Shrink), cập nhật translated_html của bảng pages và chuyển trạng thái tài liệu thành compiled.
    • Pha 6 (Compiled Delivery): Gọi GET /api/docs/{doc_id}/pages/{page_num} để lấy mã HTML dịch hoàn chỉnh, trả về phía client-side để hiển thị tuyệt đối.

2. 3 Trạm Kiểm soát Chất lượng (Quality Gates)

Hậu kiểm tự động trước khi đóng gói sản phẩm tĩnh:

  • Quality Gate 1 (Encoding Gate): Xác thực tệp dịch không trống và chuẩn hóa bộ mã ký tự sang UTF-8 để hiển thị dấu tiếng Việt chuẩn xác.
  • Quality Gate 2 (DOM Structural Gate): Đối chiếu số lượng thẻ <span> giữa file Anh và file Việt. Mọi sai số (do AI dịch sót) sẽ bị báo đỏ và chặn biên dịch trang lỗi.
  • Quality Gate 3 (Overlap Gate): Quét tọa độ top/left để cảnh báo sớm nguy cơ đè chữ.

3. Quy trình Bẫy lỗi & Tự Phục hồi Động

Bẫy lỗi runtime khi tương tác dịch thuật ngoại vi hoặc render client:

  • Bẫy lỗi Lệch phân mảnh API: Khi split tag <hr> nếu số lượng spans lệch, hệ thống tự động bẫy lỗi và chuyển sang cơ chế Dịch đơn lẻ từng đoạn (Paragraph Fallback) để bảo toàn trang.
  • Bẫy lỗi Cú pháp JSON Locale: Bắt lỗi cú pháp JSON và tự Regex sửa đổi. Nếu irreparable, tự động fallback hiển thị chữ gốc tiếng Anh kèm ký hiệu cảnh báo [?] để tránh làm treo trình đọc.
  • Bẫy lỗi Tràn trang di động (LSE Overflow): Nếu chữ Việt quá dài vượt canvas $1188\text{px}$, client-side tự động giảm tỷ lệ font-size $5\%-10\%$ toàn trang (Dynamic Font Shrink) để khít khung canvas di động.

4. Cơ chế Thông dịch Chất lượng cao (Semantics Preservation)

Đảm bảo việc dịch thuật đa ngôn ngữ trôi chảy và bảo toàn 100% ý nghĩa gốc:

  • Context-Aware Reconstruction: Tự động gom các thẻ span cận kề thành câu hoàn chỉnh, tránh dịch vụn vặt làm mất ngữ cảnh. Dùng các mã định danh ẩn tinh gọn để ánh xạ 1-1 chính xác khi tái dựng HTML.
  • Agentic Multi-Stage Workflow: AI dịch thuật 3 bước khép kín (Dịch thô sát nghĩa -> Đánh bóng văn phong kỹ thuật -> Đối chiếu song song phát hiện và tự sửa sai lệch ngữ nghĩa).
  • Glossary-Enforced Injections: Áp dụng bảng từ điển thuật ngữ cốt lõi chuyên ngành để giữ nguyên giá trị tài liệu kỹ thuật gốc, ngăn việc dịch sai hoặc địa phương hóa quá mức.

4. Trình đọc Static siêu nhẹ & DRY Navigation

Ứng dụng phía client tuân thủ nghiêm ngặt nguyên lý YAGNI, sử dụng HTML5 tĩnh và Vanilla Javascript thuần túy. Hệ thống hoàn toàn không sử dụng các framework nặng nề (như React, Vue, Next.js) hay trình quản lý trạng thái phức tạp, giúp trang tải tức thì:

  • YAGNI Client Ebook Reader: Chuyển đổi ngôn ngữ siêu tốc bằng cách hoán đổi thuộc tính `src` của thẻ `<iframe>` sang thư mục ngôn ngữ tương ứng (`pages/` hoặc `pages_vi/`), tận dụng bộ đệm (cache) tự nhiên của trình duyệt mà không cần chạy code JavaScript biên dịch phức tạp.
  • DRY State Machine (js/app.js): Toàn bộ trạng thái thu phóng, ngôn ngữ, chương mục và trang hiện tại được quản lý tập trung trong một đối tượng `state` duy nhất. Không trùng lặp các hàm tính toán vị trí hay cập nhật giao diện.
  • Seamless Page Scrolling: Cơ chế phát hiện điểm cuối trang thông minh tự động tải trang kế tiếp khi người dùng cuộn chuột hoặc vuốt chạm, tạo cảm giác đọc liền mạch như ứng dụng bản địa.

import os

def verify_raw_assets(pages_dir, total_expected=482):
    print("Khởi chạy pipeline kiểm tra chất lượng tài nguyên tĩnh...")
    missing_pages = []
    
    for i in range(1, total_expected + 1):
        page_path = os.path.join(pages_dir, f"Agentic_Design_Patterns-{i}.html")
        if not os.path.exists(page_path):
            missing_pages.append(i)
            continue
            
        # Kiểm tra tính toàn vẹn của tệp tĩnh

    else:
        print(f"Lỗi: Thiếu các trang: {missing_pages}")

// 2. PYTHON BEAUTIFULSOUP (extract_to_json.py): Bóc tách text span giữ nguyên index (DRY)
import os, json
from bs4 import BeautifulSoup

def extract_spans_to_json(html_path, json_output_path):
    with open(html_path, 'r', encoding='ISO-8859-1') as f:
        soup = BeautifulSoup(f.read(), 'html.parser')
    
    # Trích xuất toàn bộ các thẻ <span> chứa chữ từ pdftohtml
    spans = soup.find_all('span')
    texts = {str(idx): span.get_text() for idx, span in enumerate(spans)}
    
    with open(json_output_path, 'w', encoding='utf-8') as f:
        json.dump(texts, f, ensure_ascii=False, indent=2)

// 3. PYTHON REGEX BUILDER (build_translated_html.py): Đồng bộ hóa ngôn ngữ giữ nguyên tọa độ absolute
import re

def build_translated_page(html_path, json_trans_path, output_path):
    with open(html_path, 'r', encoding='ISO-8859-1') as f:
        html_content = f.read()
    with open(json_trans_path, 'r', encoding='utf-8') as f:
        translations = json.load(f)

    class SpanReplacer:
        def __init__(self, translations):
            self.idx = 0
            self.translations = translations
        def __call__(self, match):
            idx_str = str(self.idx)
            self.idx += 1
            opening, content, closing = match.group(1), match.group(2), match.group(3)
            # Tráo nội dung dịch nếu tồn tại, giữ nguyên toàn bộ CSS inline style của thẻ span gốc!
            return f"{opening}{self.translations[idx_str]}{closing}" if idx_str in self.translations else match.group(0)

    replacer = SpanReplacer(translations)
    # Sử dụng Regex thế chỗ nội dung text span cực nhanh
    new_html = re.sub(r'(<span[^>]*>)(.*?)(</span>)', replacer, html_content, flags=re.I|re.S)
    
    with open(output_path, 'w', encoding='utf-8') as f:
        f.write(new_html)

function handleWheel(e) {
    const container = viewerContainer;
    const atBottom = container.scrollTop + container.clientHeight >= container.scrollHeight - 2;
    const atTop = container.scrollTop <= 2;
    
    if (e.deltaY > 0 && atBottom) {
        // Cuộn xuống cuối -> Chuyển sang trang sau
        loadPage(currentPage + 1, false); // Tải đầu trang sau
    } else if (e.deltaY < 0 && atTop) {
        // Cuộn lên đỉnh -> Quay lại trang trước
        loadPage(currentPage - 1, true); // Tải cuối trang trước
    }
}

# 1. THUẬT TOÁN TÁI LẬP NGỮ CẢNH (HFE Context Reconstruction Python Spec)
def reconstruct_context_and_index(spans_list, threshold_y=5.0):
    # Sắp xếp span theo dòng (tọa độ y) và left
    spans_list.sort(key=lambda x: (x['top'], x['left']))
    groups = []
    current_group = []
    
    for span in spans_list:
        if not current_group:
            current_group.append(span)
        else:
            if abs(span['top'] - current_group[0]['top']) < threshold_y:
                current_group.append(span)
            else:
                groups.append(current_group)
                current_group = [span]
    if current_group:
        groups.append(current_group)
        
    reconstructed_blocks = []
    for group in groups:
        group.sort(key=lambda x: x['left'])
        text_parts = [f"[s:{s['index']}] {s['text']}" for s in group]
        reconstructed_blocks.append(" ".join(text_parts))
    return reconstructed_blocks

// 3. DYNAMIC FONT SHRINK CLIENT-SIDE (Safeguard JavaScript Recovery)
function adjustFontSizeToFit(iframe, maxHeight = 1188, step = 0.02, minScale = 0.75) {
    const doc = iframe.contentDocument || iframe.contentWindow.document;
    const body = doc.body;
    
    if (!body.style.fontSize) {
        body.style.fontSize = "100%";
    }
    let currentScale = 1.0;
    
    function getAbsoluteBoundingBottom() {
        let maxBottom = 0;
        const elements = body.querySelectorAll('div, span, p');
        elements.forEach(el => {
            const rect = el.getBoundingClientRect();
            const bottom = (el.offsetTop || 0) + (el.offsetHeight || rect.height || 0);
            if (bottom > maxBottom) {
                maxBottom = bottom;
            }
        });
        return maxBottom;
    }
    
    let realBottom = getAbsoluteBoundingBottom();
    while (realBottom > maxHeight && currentScale > minScale) {
        currentScale -= step;
        body.style.fontSize = `${currentScale * 100}%`;
        if (typeof window.applyLayoutShiftEngine === 'function') {
            window.applyLayoutShiftEngine(doc);
        }
        realBottom = getAbsoluteBoundingBottom();
    }
    return {
        finalBottom: realBottom,
        fittedScale: currentScale,
        overflowRecovered: realBottom <= maxHeight
    };
}
